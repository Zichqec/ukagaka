Permissions

This page outlines permissions for my various works. Please refer to it when using my works!

A couple of broad notes first:

  • For assets which require credit, I can be credited as "Zichqec", with the following link: https://ukagaka.zichqec.com/

  • Some works may have specific permissions. Please check the listings/readmes for each work, and follow the specific permissions if any are given.

  • Some of the works listed on this website are collaborative. My collaborators may have different permissions than me. Please be sure to respect the permissions given by my collaborators if they are more restrictive, and if you're uncertain of the permissions, please ask us.

  • I may update this list from time to time. It may be a good idea to check it immediately before using one of my assets.

  • If you're reading this page through machine translation, here is an explanation of the terms that will be used for sections about using free assets:

    • "You must" - These are things you must do when using one of my works.

    • "You may" - These are things that you can optionally do when using one of my works.

    • "You must not" - These are things you must not do when using one of my works.

You'll find a few common themes among my permissions. I'll list them here as well, as permissions that generally apply to everything I make.

  • I only allow redistribution and modification for certain types of works. Otherwise, redistribution and modification are not permitted.

    • If you wish to create a translation for one of my works, please contact me for permission first.

  • I do not want my work used for commercial purposes, especially my drawings.

  • I provide free-to-use assets for the exclusive use of the ukagaka community, and do not want them used outside of this context. (That means they cannot be used for things such as Shimeji, video streaming, website decorations, etc., without express permission.)

    • Code assets (including SAORI and tools) are exempt from this. If you can find a use for them in a non-ukagaka setting, you may use them.

  • I do not want my work turned into anything discriminatory or political, or used in a project that contains these elements.

  • I do not want my work modified to add nudity or sexual elements.

  • I do not want my work modified by generative AI tools, or used as training data for generative AI.

Besides this, I also find it frustrating when something I've created has my name stripped off of it and then is spread around. This has happened a few times and is demoralizing. Please take care to credit appropriately, especially if you are making a ghost for a popular fandom.

If there's anything on this page that you're uncertain of, get in contact and I'll clarify!

Ghosts

This section uses a translated and adapted version of the "ghost copyright awareness" list on the specwiki.

About secondary works (fanart, etc.)

  • [1. Producing and distributing additional shells, balloons, etc.]

    • OK - But making additional shells is not recommended for ghosts with very specialized shells where more content may be added over time (such as KEEP FROZEN).

  • [2. Creating secondary works.]

    • OK for visual content such as drawings, animations, etc.

    • NG for written content. This includes visual content that has dialogue, such as long comic strips.

  • [3. Secondary works that contain NSFW/R-18/R-18G content (sexual content or gore).]

    • NG - However, note that moderate injury and blood does not fall under this category and is therefore allowed.

  • [4. Depicting the ghost in secondary works that are far removed from the original setting.]

    • OK - Original reimaginings are welcome. However, please take great care when representing my characters as secondary works for another media. I would prefer this not be done in a lot of cases.

      • It can be hurtful when it comes across as someone taking my original work and making it more palatable for themself by changing it to a media they are familiar with. Please consider whether what you are making is something you know that I or my collaborators will enjoy, or whether it is made for your own enjoyment.

  • [5. Selling secondary works.]

    • NG

  • [6. Asking for ghost-related works to be created through commission services, etc.]

    • NG when requesting that others make commissions related to my works.

    • OK when requesting commissions from me, but it is unlikely I will accept.

Handling of ghosts and other distributed data

  • [1. Linking directly to the ghost download page or archive.]

    • Please link to the ghost's page on my website. If no page is available, you may link to the GitHub Releases page for the ghost. If neither is available, please contact me.

  • [2. Registering the ghost on external sites such as information aggregation sites (Ghost Captor Sakura, etc.).]

    • OK

  • [3. Redistributing ghosts that are currently available or have been discontinued.]

    • NG

  • [4. Redistributing ghosts after modifying their contents (dictionary, shell, etc.)]

    • NG - However, some translation work may be permitted. Please contact me for permission first.

  • [5. Producing/distributing additional shells that use the materials of the master shell.]

    • NG - However, in cases where I have used a freeshell, you may use it provided you follow the guidelines of the freeshell.

Other

  • [1. Giving critical feedback about ghosts.]

    • OK within reason.

  • [2. Ghost-related feature requests.]

    • OK - You may make reasonable requests, but they may not be implemented.

  • [3. Revealing spoilers in public.]

    • OK - Please mark them with a spoiler warning if you can.

  • [4. Publicly sharing quotes from dialogues or screenshots that are taken.]

    • OK

  • [5. Using shells, balloons, and other images as materials (for SNS icons, etc.)]

    • NG - However, if the ghost is made using a freeshell, then the permissions are that of the freeshell.

  • [6. Showing ghosts within videos.]

    • OK - But please do not make monetized videos centered around my ghosts. A mention or brief showing of my ghost in a monetized video is fine, but making money from a video in which my ghost is the main focus is not.

  • [7. Modifying any ghost materials (images, writing, code) with generative AI tools, or using them as training data for generative AI.]

    • NG

Shells

Note: This section only applies to works listed on the "Shells" page of my website, and not to the shells listed on the "Freeshells" page.

These are shells which are created for specific ghosts, rather than shells to create your own works with.

I do not allow these shells to be modified or redistributed.

Permissions for these shells are the same as for my ghosts, except in cases where the permissions of the ghost that the shell is made for are stricter. Please be sure to check the ghost's permissions as well, especially if I am not the author of the ghost.

If I have created a shell for your ghost

  • Feel free to add custom dialogue or behavior to your ghost based on the shell.

  • Feel free to list my shell on your ghost's website page, etc. Please be sure to link to the shell's page on my website and not a direct download. If no page is available, please contact me.

  • Please do not bundle it with the ghost's nar file unless I have given express permission to do so. I'd prefer that my shells be obtained separately!

    • This also applies to using commands in SSP to automatically install the shell by downloading it from the internet, etc. I would like the user to be directed to my website to obtain any shells I make, so that they can see it in full context.

Freeshells

Note: This section only applies to works listed on the "Freeshells" page of my website, and not to the shells listed on the "Shells" page.

These are shells to create your own ukagaka works with. They are free to use with the following terms:

You must

  • Mention in your ghost's readme that you are using my freeshell. I can be credited as "Zichqec", with the following link: https://ukagaka.zichqec.com/

  • Any modifications made to the content of the freeshell should be noted in your ghost's readme.

  • Keep the readme file that is included with the freeshell as-is. It can be renamed if needed (such as if combining two freeshells together).

  • Keep the craftman information of the freeshell as-is.

You may

  • Modify the shells, including recoloring, drawing new expressions, combining expressions, scaling the images, mirroring the images, etc. (If the shell is drawn in a pixel style, please take care when scaling. They'll look best if you keep the pixels sharp!)

  • Modify the settings files in any way you need.

  • Combine them with other freeshells.

  • Use the shell to create a thumbnail image for the ghost, or a preview image for use on the ghost's release page, etc.

You must not

  • Distribute any version of these freeshells on their own.

  • Use these freeshells to create commercial works.

  • Use these freeshells for works outside the realm of ukagaka.

  • Modify these freeshells to be discriminatory or political, or use them in works that contain these elements.

  • Modify these freeshells to add nudity or sexual elements.

  • Use AI tools to modify the artwork in these freeshells.

In the event that a particular freeshell specifies different terms in its readme file, that freeshell's specific permissions take precedence. I would appreciate if you ensure that you are using the latest version of the freeshell, in case the terms have updated.

You don't have to notify me if you use my freeshells, but if you do I would love to see what you've made!

Balloons

These permissions pertain to bundling one of my balloons with one of your ghosts. I allow my balloons to be redistributed when bundled with a ghost, but otherwise, please do not redistribute them.

You must

  • Mention in your ghost's readme that you are including my balloon with the installation. I can be credited as "Zichqec", with the following link: https://ukagaka.zichqec.com/

  • Any modifications made to the content of the balloon should be noted in your ghost's readme.

    • If you have made modifications, please change the name of the folder the balloon will be installed to in install.txt, to avoid overwriting the original version. You might consider a format such as balloonname_ghostname.

  • Keep the readme file that is included with the balloon as-is. It can be renamed if needed (such as if combining two balloons together).

  • Keep the craftman information of the balloon as-is.

You may

  • Make modifications to these balloons. Modifications include but are not limited to:

    • Font changes

    • Font size changes

    • Hue shifting / changing the colors

    • Changing the text colors

    • Adding or removing a tail

    • Setting up specific balloon colors / designs for the sakura / kero / extra characters

    • Changing various markers

    • Combining different balloons into one

You must not

  • Distribute any version of these balloons on their own.

  • Distribute a modified version of these balloons with a commercial work. (Unmodified versions are permitted.)

  • Modify these balloons to be discriminatory or political, or include them in works that contain these elements.

  • Modify these balloons to have nudity or sexual elements.

  • Use AI tools to modify the artwork in these balloons.
